Define inputs and outputs

Pin(s)/signal(s) must be mapped in InSync in order for data to be received and in order for calls (outputs) to be placed correctly. To configure the inputs and outputs in InSync:

1.

Click the InSync Mapping button on the IO Board Tester dialog box. The Edit Pin Assignments dialog box displays.

2.

Select the correct method of interfacing with the cabinet from the InSync Cabinet Interface drop-down menu.

Note:

If creating a new configuration, it may be a good idea to click the Load Defaults button or Clear All Data button.

3.

Each phase is represented by a tab on the dialog box. Select the phase you want to configure by clicking the tab.

4.

Click inside the appropriate functionality box to assign a pin to that functionality. The Select Pins dialog box displays.

5.

Select the pins that you want mapped.

Note:

The Function column in the Select Pins dialog box describes the default functionality of a pin; however, this description may not accurately reflect the functionality of your pin in your cabinet if you have set up any customization.

For example, to assign that Phase 2 should send a vehicle call on Pins Det05 and Det25 when interfacing with the cabinet using a SLDC Intercept Module, perform the following steps:

1.

Select SDLC Intercept Module from the InSync Cabinet Interface drop-down box.

2.

Click the Phase 2 tab.

3.

Click in the Vehicle Ext/Call field. The Select Pins dialog box displays.

4.

Click Det05 and scroll down and click Det25.

5.

Click OK. The pins are shown in the Vehicle Ext/Call field.
